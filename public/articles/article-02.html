设计实践 AI 方法论

AI 重塑设计：设计工作的智能化演进实践

来自阿里云政企智算域设计团队的内部分享摘要 · 人名已脱敏处理 · 原文含图示与演示视频

💡核心观点

设计师不会被 AI 取代，而是能借助 AI 焕发新活力——核心在于既深度洞察客户与用户诉求，又熟悉新技术迭代并善用其赋能业务场景，逐步从单一能力设计师进化为新时代的超级个体。

智能化能力演进历程

  • 阿里云政企设计团队在过去十年中经历了从人工智能初步发展、大模型生成式多模态，到 2024 年智能体爆发与行业深度落地的完整演进，服务涵盖城市大脑、辅助医疗等前沿领域
  • 技术端拓展出 LoRA 训练、3D 生成、Agent 智能体等能力，应用端覆盖品牌、游戏、金融、法律、企业管理等广泛产业场景，AI 技术从基础工具向产业级协同系统进阶
  • 团队将每年涌现的新技术提炼为「智能原子能力卡」，帮助设计师与客户沟通时快速选型，并在方案中实现技术与场景的精准连接与落地

智能化价值评估体系

  • 在传统行业生产工作流中引入「脑机比」概念，量化人工与 AI 各自的投入比重，客观评估每个环节的智能化改造收益，避免盲目追求智能化
  • 建立智能化价值评估象限：横轴为技术实现复杂度，纵轴为场景价值（影响力大小），将阿里云设计工作的所有场景（用研、项目咨询、客户交付、产品设计、展会、品牌等）纳入四象限，优先聚焦高价值场景

典型落地案例

  • 营销长图智能化（钉钉卡片方案）：针对运营长图修改频次高、沟通成本大的痛点，利用钉钉卡片的 JSON 代码本质，构建代码转换 Agent，将设计内容自动转化为可点击的钉钉运营卡片。方案轻量、复用性强，且投放后所有点击数据均可追踪回溯，解决了运营内容价值难以量化的问题
  • 产销培训视频生成（数字人方案）：针对产品出海宣贯、多语言培训内容制作需求，调研市面上多款 MCP 能力后，将性价比最优方案封装至自研 Agent，实现声纹 + 形象采集后自动生成数字人视频。引入并行渲染逻辑后，生成时间从 3 小时缩短至 30 分钟
  • 鞋履设计全流程智能化（闭源大模型方案）：服务某运动领域品牌客户，梳理其企划需求→鞋型发散→配色→定版全流程，基于客户沉淀的十余万张高质量鞋履设计图片，构建行业专属闭源大模型，实现 AI 辅助覆盖鞋履生产全链路

今年云栖大会上发布的 ArtWeaver 平台，已服务阿里云 50 余个业务线及海量外部客户，包含大量 AIGC 类内容，可大幅提升日常设计工作效率，并为客户沟通展示提供丰富素材支持。

智能化设计咨询方法论

  • 阿里云设计团队提炼出一套完整的智能化设计咨询流程，帮助客户完成从业务现状梳理到 AI 转型蓝图的全链路交付，可推广、可复用

四步咨询流程

步骤 核心方法 产出物
① 场景调研 三维要素调研法：角色 · 流程 · 触点，深度洞察业务现状与客户核心痛点 业务现状流程图
② 场景评估 五维评估法：需求强度 / 大模型匹配度 / 技术可实现度 / 盈利潜力 / 场景孵化潜力 优质场景清单（含五维度打分）
③ 场景构建 基于 5 阶段智能化模型（S1→S5），将大模型能力与现有业务深度融合，绘制未来业务流程图并标注 AI 介入点 AI 赋能后的未来业务流程图
④ 设计演示 三种交互模式（对话式 · 辅助式 · 嵌入式）搭配横纵坐标评估体系，用 UI 界面 + 演示视频呈现未来业务形态 可视化原型 + 场景化脚本

场景调研：三维要素法

  • 调研是一切的根基。设计师来自设计背景而非业务背景，必须在帮助业务进行大模型转型之前，深入了解业务现状，洞察客户真实痛点

三维调研要素

◉ 角色维度 识别场景中存在的关键业务人员类型及其核心职责
⇢ 流程维度 还原不同业务人员在实际工作中的配合关系与全流程节点
⬡ 触点维度 梳理业务系统、线上沟通等各类交互触点，判断 AI 切入可行性

场景构建：5 阶段智能化升级路径

  • 基于人与 AI 的占比关系，将业务场景智能化程度划分为 5 个阶段，AI 占比逐级提升、人工参与逐级降低。目前大多数项目处于 S3–S4 阶段，S5 为未来目标形态

场景智能化阶梯

S1
在线办公 业务全面数字化，信息可在线流转，具备智能化升级基础
S2
信息咨询 AI 回答知识性问题，人向 AI 提问、AI 给出答案
S3
辅助决策 AI 提供数据分析与方案建议，人工负责最终决策判断
S4
自主执行 Agent 自主完成特定繁琐事务，人工专注监督与确认
S5
全自动化 端到端无人工干预，AI 全流程闭环运转，为未来目标形态

数字员工 Agent 协同体系

  • 以智能体为核心，构建由 5 个专项 Agent 组成的全流程协同方案：输入客户访谈资料后，几分钟内可自动生成结构清晰、逻辑完整的行业智能化转型报告

5 Agent 协同链路

Agent 1
业务梳理 Agent
解析客户访谈资料，基于预制业务流程图映射机制，生成完整的业务现状流程图，将抽象文档转化为直观可视化呈现
Agent 2
场景筛选 Agent
对业务流程图进行交叉验证，依托大模型五维评估法（需求强度 / 匹配度 / 可实现性 / 盈利潜力 / 孵化潜力）综合打分，输出含分级的优质场景清单
Agent 3
技术方案 Agent
基于接入的阿里云产品能力库，为优质场景匹配技术路径，分析实现方式，输出含优势与实施难度的推荐技术方案，助力交付团队判断可行性
Agent 4
流程重构 Agent
基于大模型场景构建评估方法，绘制未来版业务流程图，清晰标注 AI 介入点与人机协同模式，输出传统流程 AI 赋能的完整升级路径
Agent 5
脚本生成 Agent
基于优化后的业务流程与 AI 嵌入等级，生成含场景背景、角色设定、人机对话及页面表现的场景化脚本，助力客户直观感知 AI 赋能后的业务未来样态

最终整合输出的完整报告涵盖：业务现状流程图 · 优质场景清单 · 技术落地方案 · 未来业务流程图 · 场景化脚本文案，形成可推广、可复用、可迭代的 AI 落地新模式。

对设计师的启示

  • 定义问题的能力：在 AI 可以执行大量生产任务的时代，设计师的核心价值在于识别和定义正确的问题，而非仅仅交付解决方案
  • 人机协作的领导力：理解 AI 的能力边界，知道何时信任、何时干预，能够指挥 Agent 完成复杂任务，成为人机协作的组织者
  • 系统性商业思维：从单一视觉执行者转变为能够理解业务价值链、推动 AI 落地产生商业价值的设计决策者

未来的设计师，是一个懂人性的逻辑学家，也是一个懂技术的艺术家。你的价值不在于 Figma 图层整理得有多干净，而在于如何驾驭 AI 这个强大的工具，去创造真正解决人类问题、且具有商业价值的产品。
