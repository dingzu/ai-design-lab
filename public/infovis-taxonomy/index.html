技术实现路径维度 · How to Build
按开发方式与工具链对图表实现路径分类。不同路径在灵活度、学习成本、定制能力、部署效率上存在显著差异。选路径前先回答：
谁来做？（工程师/设计师/分析师） ·  多复杂？（标准图 vs 高度定制） ·  多实时？（静态 vs 动态数据）
PATH 01
原生代码 Native Code (SVG / Canvas / WebGL)
直接操作浏览器底层渲染 API，最高灵活度，适合深度定制与超大数据量场景
3 种技术栈
📐 SVG
Scalable Vector Graphics
基于 XML 的矢量图形标准，DOM 可操控，天然支持无限缩放与 CSS 动画。适合节点数 < 10,000 的精细图表。
原生 SVG D3.js Snap.svg GSAP
适合图型
柱状图折线图散点图 力导向图桑基图树状图 雷达图旭日图
优势
  • 无限矢量缩放
  • DOM 可访问性好
  • CSS/JS 动画自由
  • 调试工具友好
    • 局限
  • 万级节点性能差
  • 学习曲线陡峭
  • 开发成本高
    • 🎨 Canvas 2D
    HTML5 Canvas 2D Context
    像素级光栅渲染，GPU 加速，大数据量下性能远优于 SVG。适合实时数据流、热力图、百万级散点等高密度场景。
    Canvas API Fabric.js Konva.js ZRender
    适合图型
    热力图大规模散点密度图 实时折线蜡烛图粒子动效
    优势
  • 百万级数据流畅
  • 实时动画性能佳
  • 像素级控制
    • 局限
  • 不可矢量缩放
  • 无 DOM，交互复杂
  • 可访问性差
    • ⚡ WebGL / GPU
    WebGL / WebGPU
    GPU 着色器级渲染，亿级数据点可视化的唯一选择。适合 3D 地理可视化、实时大屏、科学可视化等极端场景。
    Three.js deck.gl Luma.gl Regl
    适合图型
    3D 地图大屏可视化亿级散点 实时流图体渲染GIS 分析
    优势
  • 极限数据量
  • 3D 原生支持
  • 实时 60fps
    • 局限
  • 门槛极高
  • 调试困难
  • 移动端功耗大

    可视化库 Chart Library / Visualization Framework

    Skill 工程化路径维度 · How to Skill
    基于本 Skill 体系对图表生成任务的工程化封装方式。核心 Skill 包括： visual-creator infovis-evaluator infovis-autoevo design-evaluator
    按任务复杂度选路径：单次生成 → 生成+评测 → 自进化闭环。越靠后质量越高，代价越大。

    单点生成 visual-creator 直调
    最轻量路径，一次 Skill 调用直接输出图表。适合需求明确、质量要求不高的快速生成场景
    适合：快速原型 / 一次性需求
