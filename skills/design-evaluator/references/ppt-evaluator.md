# PPT / 演示文稿专项评测规则

## P1 — 叙事结构 (0–10分)

### 评分项

#### P1.1 — 故事弧完整性 (0–4分)

优秀的演示文稿遵循"情境→冲突→解决方案→行动"结构（SCQA框架）：

| 得分 | 条件 |
|------|------|
| 4 | 有明确的情境建立、问题/冲突揭示、解决方案展示、行动号召 |
| 3 | 有结构但某一环节弱（如结尾缺少行动号召） |
| 2 | 有隐含结构，但需要受众主动推断 |
| 1 | 内容堆砌，无叙事逻辑 |
| 0 | 无法识别叙事意图 |

常见结构模板：
```
[开场] 情境建立: "我们处于什么背景下..."
[冲突] 问题揭示: "但目前面临的挑战是..."
[转折] 洞察/解法: "我们发现/提出..."
[收尾] 行动号召: "因此我们建议/需要决策..."
```

#### P1.2 — 开场钩子强度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 开场直接触达受众最关心的问题或痛点 |
| 2 | 开场有引入，但不够紧迫或有说服力 |
| 1 | 开场是议程页或自我介绍，延迟进入正题 |
| 0 | 开场是封面页+目录，完全没有钩子 |

#### P1.3 — 结尾行动清晰度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 明确的 Next Step：谁在何时做什么决策/行动 |
| 2 | 有总结，但行动项模糊 |
| 1 | 仅有"谢谢"页或联系方式 |
| 0 | 无结尾，内容突然停止 |

## P2 — 单页原则遵守度 (0–10分)

### "每页一个观点"原则

#### P2.1 — 标题即观点 (0–4分)

优秀标题 = **行动/观点句**（而非主题词）：

| 类型 | 示例 | 评价 |
|------|------|------|
| ✅ 观点句 | "电商部门 Q4 满意度跌破预警线，需立即干预" | 读标题即知结论 |
| ⚠️ 主题词 | "电商部门 Q4 满意度分析" | 只知道讲什么，不知道结论 |
| ❌ 功能词 | "数据" / "背景" / "第二部分" | 完全无信息量 |

评分方式：统计全部 slide 标题，计算"观点句"比例：
- 80%+ → 4分
- 60–79% → 3分
- 40–59% → 2分
- 20–39% → 1分
- < 20% → 0分

#### P2.2 — 单页信息密度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 每页核心信息点 ≤ 3，有充足留白 |
| 2 | 每页 4–5 个信息点，略显拥挤 |
| 1 | 每页 6–8 个信息点，难以快速消化 |
| 0 | "文字墙"：整页密密麻麻，无视觉呼吸 |

#### P2.3 — 正文支撑标题 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 正文内容直接支撑标题观点，无跑题 |
| 2 | 正文大部分相关，有少量无关信息 |
| 1 | 正文扩展了多个方向，稀释了标题焦点 |
| 0 | 正文内容与标题无关或矛盾 |

## P3 — 演讲/阅读模式适配 (0–10分)

### 判断目标模式

首先确认PPT的使用场景：

| 场景 | 特征 | 设计要求 |
|------|------|---------|
| **演讲辅助型** | 演讲者主导，slides 辅助 | 精简文字，视觉冲击，配合口述 |
| **自读文档型** | 受众独立阅读，无演讲者 | 文字完整，逻辑自洽，无需解释 |
| **双用途型** | 既要演讲又要发送传阅 | 最难平衡，通常需要两套版本 |

#### P3.1 — 模式一致性 (0–4分)

| 得分 | 条件 |
|------|------|
| 4 | 全程保持一种模式，设计选择支持该模式 |
| 3 | 基本一致，有少量混用 |
| 2 | 模式混乱：有些页过于精简（演讲型）、有些页过于密集（文档型） |
| 0 | 典型"双败"：文字多到演讲时受众在读slide，少到阅读时缺乏上下文 |

**演讲辅助型** 质量标准：
- 每页文字 ≤ 30 字（正文）
- 大图 / 单一数据 / 一句核心词
- 不依赖 slide 传递完整逻辑，口述补充

**自读文档型** 质量标准：
- 每页信息自洽，无需外部解释
- 术语有定义，数据有上下文
- 建议阅读时间与页数匹配

#### P3.2 — 缺陷诊断

常见错误模式：

```
❌ "文字墙" 型: 把 Word 报告内容直接复制进 slide
   → 修复: 提炼为 3 个关键点，详情放备注页

❌ "纯图片" 型: 演讲型PPT发给无法参会的人
   → 修复: 制作配套的"阅读版" (附上演讲稿或备注)

❌ "标题党" 型: 每页标题是观点，但正文只有一句话
   → 修复: 添加支撑数据/例证，或合并相似页
```

#### P3.3 — 备注页质量 (0–2分)

| 得分 | 条件 |
|------|------|
| 2 | 演讲型PPT有详细备注，自读型无需备注 |
| 1 | 有备注但信息量不足或与slides内容重复 |
| 0 | 演讲型PPT无备注，信息完全依赖演讲者记忆 |

## PPT 专项综合分计算

```
P1(叙事结构)    = P1.1×0.4 + P1.2×0.3 + P1.3×0.3    [0–10]
P2(单页原则)    = P2.1×0.4 + P2.2×0.3 + P2.3×0.3    [0–10]
P3(模式适配)    = P3.1×0.6 + P3.2评估 + P3.3×0.2     [0–10]

PPT专项分 = (P1 + P2 + P3) / 3                         [0–10]
```

## PPT 变异建议模板

### 叙事结构修复

```
当前问题: [缺失冲突/开场无钩子/结尾无行动项]

修复建议:
  Slide 1: 替换封面为"问题陈述页"
    标题: "[用户最关心的问题] — 我们如何解决？"
    
  最后一页: 替换"谢谢"为"下一步行动"
    内容: 
    □ [责任人A] 在 [日期] 前完成 [具体行动]
    □ 今天需要决策的是：[具体决策]
```

### 标题观点化

```
原标题: "用户调研分析"
→ 改为: "73% 用户因响应速度不满意——移动端优先级需提升"

原标题: "技术方案对比"
→ 改为: "方案 B 在成本和时间上均优于方案 A，推荐采用"
```

### 文字密度修复

```
当前: Slide 包含 12 个 bullet points
修复步骤:
  1. 识别最核心的 1 个观点 → 放标题
  2. 最重要的 2–3 个支撑点 → 保留正文
  3. 其余内容 → 移至备注页 或 创建附录 slide
```
