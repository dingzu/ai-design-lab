# 报告/文档专项评测规则

## R1 — 摘要质量 (0–10分)

### 评分项

#### R1.1 — 执行摘要完整性 (0–4分)

优秀的执行摘要包含三要素：**背景 + 核心发现 + 行动建议**。

| 得分 | 条件 |
|------|------|
| 4 | 摘要包含背景、发现、建议三要素，仅读摘要即可做出决策 |
| 3 | 包含两要素（如有背景+发现，但建议模糊） |
| 2 | 摘要存在但只描述问题/现状，无结论或建议 |
| 1 | 有摘要段落，但内容与报告正文高度重复，无提炼 |
| 0 | 无执行摘要，报告直接从背景或正文开始 |

**摘要质量模板**：
```
[背景] 在 [情境] 下，本报告分析了 [主题]。
[发现] 核心发现：[2-3 个最重要结论]。
[建议] 我们建议 [具体行动]，预计 [量化影响]。
```

#### R1.2 — 结论前置度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 每章/每节开头都有该节结论，读者无需通读全节 |
| 2 | 主要章节结论前置，部分章节需读完才能知道结论 |
| 1 | 结论在章节末尾，遵循"起因→过程→结论"的传统叙事顺序 |
| 0 | 全文无结论，只有描述和数据，依赖读者自行提炼 |

#### R1.3 — 摘要与正文一致性 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 摘要中的结论与正文完全对应，数字/表述一致 |
| 2 | 摘要与正文基本一致，有轻微表述差异 |
| 1 | 摘要结论与正文细节存在出入，需要仔细核对 |
| 0 | 摘要内容与正文严重不符，或摘要包含正文中未出现的结论 |

---

## R2 — 结构导航 (0–10分)

### 评分项

#### R2.1 — 目录与层级体系 (0–4分)

| 得分 | 条件 |
|------|------|
| 4 | 有目录，标题层级 ≤ 3 级，各级标题遵循"观点句"原则 |
| 3 | 有目录，但标题多为"主题词"（如"第三章：数据分析"），缺少结论信息 |
| 2 | 有目录，但层级混乱（如直接从一级标题跳到三级） |
| 1 | 无目录，但文档有明显的标题体系 |
| 0 | 无目录，无标题层级，整文为流水段落 |

**观点句标题示例**：
| 类型 | 示例 | 评价 |
|------|------|------|
| ✅ 观点句 | "用户满意度下降源于响应速度问题，需优先优化" | 读标题即知结论 |
| ⚠️ 主题词 | "用户满意度分析" | 只知道讲什么，不知道结论 |
| ❌ 功能词 | "第三节" / "背景" | 无信息量 |

#### R2.2 — 章节过渡质量 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 长文档（>5页）每章结尾有简短过渡语，引导进入下一章 |
| 2 | 有部分过渡，整体逻辑连贯但偶有跳跃感 |
| 1 | 章节之间无过渡，读者需自行理解逻辑关系 |
| 0 | 短文档无此项要求（≤3页，得分默认 3 分） |

#### R2.3 — 引用与来源规范 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 所有数据/引用都有来源标注，来源格式统一 |
| 2 | 大部分数据有来源，少量数据未标注 |
| 1 | 仅部分数据有来源，标注不规范或格式混乱 |
| 0 | 无任何来源标注，数据可信度无法核实 |

> 注：内部报告（无需引用外部来源）时，此项评分以"数据标注的完整性"（单位/时间/口径）替代。

## R3 — 数据呈现质量 (0–10分)

### 评分项

#### R3.1 — 图表化率 (0–4分)

| 得分 | 条件 |
|------|------|
| 4 | 核心数据全部以图表形式呈现，纯文字叙述仅用于背景/说明 |
| 3 | 主要数据有图表，但部分关键数据仍以数字表格呈现 |
| 2 | 有少量图表，但多数数据埋在段落文字中 |
| 1 | 极少图表（1-2 个装饰性图），数据主要靠文字传达 |
| 0 | 全文无图表，纯文字数字报告 |

#### R3.2 — 图表自读性 (0–3分)

每个图表应能独立理解，无需阅读正文：

| 得分 | 条件 |
|------|------|
| 3 | 每张图表都有：① 标题（含结论）② 轴标签/图例 ③ 数据来源/时间范围 |
| 2 | 图表有标题和轴标签，但缺少来源或时间信息 |
| 1 | 图表有标题，但标题是主题词而非结论（如"销售趋势图"） |
| 0 | 图表无标题或无轴标签，需正文解释才能理解 |

**自读性图表标题示例**：
```
❌ 差: "销售数据图"
⚠️ 一般: "Q4销售趋势（2024）"
✅ 好: "Q4销售额同比增长23%，电商渠道贡献主要增量（2024Q4 vs 2023Q4）"
```

#### R3.3 — 数字上下文完整性 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 所有关键数字都有：单位 + 时间范围 + 对比基准（环比/同比/目标值） |
| 2 | 有单位和时间，但缺少对比基准（如"收入 1000 万"而非"收入 1000 万，同比+15%"） |
| 1 | 有单位，无时间范围，数字的意义依赖背景知识 |
| 0 | 数字无单位、无时间、无上下文，读者无法判断好坏 |

## Report 专项综合分计算

```
R1(摘要质量)    = R1.1×0.4 + R1.2×0.3 + R1.3×0.3    [0–10]
R2(结构导航)    = R2.1×0.4 + R2.2×0.3 + R2.3×0.3    [0–10]
R3(数据呈现)    = R3.1×0.4 + R3.2×0.3 + R3.3×0.3    [0–10]

Report专项分 = (R1 + R2 + R3) / 3                      [0–10]
```

## Report 变异建议模板

### 摘要修复

```
当前问题: 摘要只描述背景，无核心发现和行动建议

修复建议:
  在摘要末尾添加：
  "综上，本报告发现 [核心结论1] 和 [核心结论2]。
   建议 [具体行动]，预计在 [时间范围] 内实现 [量化目标]。"
```

### 标题观点化

```
原标题: "第三章：用户行为分析"
→ 改为: "移动端用户留存率下降13%——功能复杂度是主因"

原标题: "竞品对比"
→ 改为: "竞品在价格和响应速度上均优于我方，需要在 Q2 做出应对"
```

### 数据图表化

```
当前: "2024年Q4销售额为1032万，Q3为891万，Q2为756万，Q1为623万"

修复步骤:
  1. 将季度数据转为折线图/柱状图
  2. 图表标题写成结论句："2024年销售额季度持续增长，Q4达全年峰值"
  3. 在正文仅保留趋势说明，删除原始数字枚举
```

### 结论前置化

```
当前结构: 
  [背景] → [方法] → [数据] → [分析] → [结论]（第5页才出现）

修复结构:
  [结论先行] → [支撑论据] → [数据证明] → [附录（详细方法）]
  
  每节开头模板:
  "本节核心结论：[结论句]。以下是支撑数据..."
```

## 常见 Report 类型特殊处理

### 学术/研究报告

- R1.3（摘要一致性）权重提升：需严格核查摘要数据与正文一致
- R2.3（引用规范）为强制项：< 2 分直接触发 P0 警告
- R3.1（图表化率）权重降低：学术报告可接受更多文字描述

### 商业/决策报告

- R1.1（执行摘要）为强制项：高管受众无摘要直接 P0
- R1.2（结论前置）权重最高：决策者时间有限，结论必须前置
- R3.3（数字上下文）严格执行：商业数字必须有对比基准

### 内部工作报告（周报/月报）

- R2.3（引用规范）可豁免：内部数据无需外部引用
- P3 维度（演讲/阅读适配，来自 PPT 规则）不适用
- 重点关注 R1.2 和 R3.3：结论是否清晰，数字是否有对比
