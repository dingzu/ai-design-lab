# 变异反馈规则与补丁格式

## 概述

变异反馈是自进化引擎的核心驱动力。本文档定义：
1. 如何将评测分数差异转化为结构化补丁
2. 补丁的优先级和执行顺序
3. 反哺给 visual-creator 的 Prompt 格式规范

---

## 四维失分映射

### 维度 1: 图表类型不匹配 (Type Mismatch)

**权重**: 20%，满分 2.0 分  
**常见失分场景**:

| 意图 | 错误选型 | 正确选型 | 原因 |
|------|---------|---------|------|
| 排名对比 | RadarChart | BarChart | 雷达图无法清晰呈现排名差异 |
| 时间趋势 | BarChart | LineChart | 柱状图趋势感弱 |
| 部分与整体 | BarChart | PieChart/Donut | 柱状图不表达占比 |
| 相关关系 | LineChart | ScatterPlot | 折线图暗示时序，而非相关 |
| 层级结构 | BarChart | TreeMap | 层级关系表达不足 |
| 多维属性对比 | PieChart | RadarChart | 饼图无法对比多维度 |

**变异补丁模板**:
```
🔧 [TYPE-FIX] 图表类型修复 (-{score_loss} 分)
  当前类型: {perceived_type}
  目标类型: {expected_type}
  选型理由: {rationale}
  生成指令: "将图表类型从 {perceived_type} 改为 {expected_type}，
             原因是 {viewer_intent} 意图下 {expected_type} 更直观"
```

---

### 维度 2: 核心洞察缺失 (Insight Gap)

**权重**: 35%，满分 3.5 分  
**常见失分场景**:

| 洞察类型 | 常见遗漏 | 修复方法 |
|---------|---------|---------|
| 预警/阈值 | 缺少阈值线 / 无颜色编码预警 | 添加 threshold_line + alert_color |
| 趋势方向 | 无标注趋势增减 | 添加趋势标注箭头 |
| 最大/最小值 | 极值未突出显示 | 高亮 max/min 数据点 |
| 异常值 | 离群点未标记 | 添加离群点标注 |
| 目标达成 | 目标线缺失 | 添加 target_line |
| 占比比较 | 占比数值未显示 | 添加百分比标注 |

**变异补丁模板**:
```
🔧 [INSIGHT-FIX] 核心洞察强化 (-{score_loss} 分)
  缺失洞察: {insight_type}
  影响: 观察者无法在 5 秒内提取核心信息
  生成指令:
    - 添加 {element_1}（如: threshold_line at value={threshold}）
    - 使用 {color_encoding}（如: 红色标记低于阈值的数据点）
    - 标注 {key_value}（如: 最低评分 2.8，高亮显示）
```

---

### 维度 3: 必要元素缺失 (Element Coverage)

**权重**: 25%，满分 2.5 分  
**必要元素检查表**:

| 图表类型 | 必要元素 |
|---------|---------|
| BarChart | 坐标轴标签、数值标签、图例（多系列时） |
| LineChart | 坐标轴标签、数据点标记、趋势标注 |
| PieChart | 比例标签、图例、标题 |
| ScatterPlot | 坐标轴标签、相关系数、参考线 |
| RadarChart | 维度标签、网格、数值刻度 |
| Heatmap | 颜色图例、坐标轴标签、数值显示 |
| 信息图 | 标题、数据来源、图例或图标 |

**变异补丁模板**:
```
🔧 [ELEMENT-FIX] 元素补全 (-{score_loss} 分)
  缺失元素列表:
    - {element_1}: {why_needed}
    - {element_2}: {why_needed}
  生成指令: "在图表中添加以下必要元素: {element_list}"
```

---

### 维度 4: 认知摩擦力过高 (Friction Reduction)

**权重**: 20%（惩罚项），摩擦力越低得分越高  
**摩擦力噪声源分类**:

#### 视觉噪声类
| 噪声源 | 摩擦贡献 | 修复指令 |
|-------|---------|---------|
| 背景网格线过密 | 高 | "减少网格线密度或去除次要网格线" |
| 装饰性背景图案 | 高 | "移除装饰性背景" |
| 3D 效果 | 中 | "改为 2D 扁平化设计" |
| 阴影/渐变滥用 | 中 | "简化阴影，使用纯色" |

#### 配色噪声类
| 噪声源 | 摩擦贡献 | 修复指令 |
|-------|---------|---------|
| 颜色数量 > 7 | 高 | "限制颜色在 5 种以内" |
| 低对比度 | 高 | "提升前景与背景对比度 ≥ 4.5:1" |
| 相邻色过近 | 中 | "相邻数据系列使用对比色" |

#### 信息密度类
| 噪声源 | 摩擦贡献 | 修复指令 |
|-------|---------|---------|
| 数据点过密 | 中 | "对大数据集使用聚合或采样" |
| 图例项目 > 8 | 中 | "合并小类别为'其他'" |
| 多重坐标轴 | 高 | "简化为单坐标轴或拆分图表" |
| 重叠标注 | 中 | "使用 leader lines 避免重叠" |

**变异补丁模板**:
```
🔧 [FRICTION-FIX] 认知摩擦减噪 (-{score_loss} 分)
  当前摩擦力: {friction_score} ({level})
  主要噪声源:
    - {noise_source_1}: 贡献 {contribution}
    - {noise_source_2}: 贡献 {contribution}
  生成指令:
    - 移除 {element_1}
    - 简化 {element_2}
    - 调整 {element_3} 颜色对比度
```

## 补丁优先级与执行策略

### 优先级矩阵

```
P0 (严重): score_loss > 2.0
  → 必须修复，否则无法收敛到 9.6
  → 生成时用【必须】强调

P1 (重要): score_loss 1.0 ~ 2.0
  → 强烈建议修复
  → 生成时用【强烈建议】强调

P2 (改进): score_loss < 1.0
  → 锦上添花
  → 生成时用【建议】标注
```

### 多补丁执行顺序

当同时存在多个补丁时，按以下顺序执行：

1. **P0 类型修复** — 先解决图表类型问题（基础正确性）
2. **P0 洞察修复** — 确保核心信息可读（最高权重维度）
3. **P1 元素补全** — 添加必要视觉元素
4. **P1/P2 摩擦减噪** — 优化视觉干净度

## 进化指令注入格式

将所有补丁合并成注入 visual-creator 的指令块：

```
=== 自进化补丁包 (v{iter}) ===

【P0 - 必须修复，严重失分】
1. [TYPE] {type_fix_instruction}
2. [INSIGHT] {insight_fix_instruction}

【P1 - 强烈建议修复】
3. [ELEMENT] {element_fix_instruction}
4. [FRICTION] {friction_fix_instruction}

【P2 - 建议优化】
5. [STYLE] {style_fix_instruction}

【禁止重复的错误】
- 禁止: {known_failure_1}
- 禁止: {known_failure_2}

【累积的成功约束（保持）】
- 保持: {successful_constraint_1}
- 保持: {successful_constraint_2}
```

## 自进化知识提炼规则

每完成一个 Case 的迭代循环后，将经验提炼存储：

### 存储条件
- **提炼时机**: Case 结束时（无论 PASS 或 FAIL）
- **存储位置**: `generator_context` 中的各知识池

### 提炼规则

```
if(某维度连续 2 次出现相同失分):
  → 加入 known_failures（永久禁止列表）

if(某修复在 1 轮内带来 > 1.0 分提升):
  → 加入 design_constraints（高效约束列表）

if(某图表类型在特定意图下反复被错误选择):
  → 加入 type_traps（类型陷阱警告）

if(某元素缺失导致 insight_blind_spot):
  → 加入 insight_blind_spots（洞察盲区）
```

## 激进变异策略

当连续 2 轮得分提升 < 0.1 分时，触发激进变异：

```
[激进变异模式]
原因: 在当前设计方向上已接近局部最优，需要突破性改变

执行:
1. 忽略所有累积的历史补丁
2. 从 Expected Topology 重新推导理想设计
3. 完全重新选择图表类型（即使之前没有类型错误）
4. 从零开始构建视觉元素层次

注入指令:
"请完全重新设计此可视化。不要参考之前的任何版本。
从数据意图 {viewer_intent} 出发，重新选择最合适的图表类型，
构建清晰的视觉层次，确保核心洞察在 3 秒内可读。"
```
