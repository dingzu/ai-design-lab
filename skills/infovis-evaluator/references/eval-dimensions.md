# 三大评测维度 — 详细评分规则

## 技能 A: 信息路由 (Semantic Router)

### 评分项

#### A1 — 图表类型匹配度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 图表类型与数据意图完全匹配（最优选择） |
| 2 | 类型基本可用，但有更优选择 |
| 1 | 类型可接受，但造成信息损失 |
| 0 | 类型严重不匹配（如用RadarChart做预警对比） |

#### A2 — 必要元素完整性 (0–4分)

对比 `required_elements` 列表，每缺少一个关键元素 -1 分，最低 0 分。

**通用必要元素**：
- 坐标轴标签（有坐标轴的图表）
- 图表标题
- 数据来源/时间范围（正式场景）
- 核心数值标注（关键数据点）

#### A3 — 意图对齐度 (0–3分)

| 得分 | 条件 |
|------|------|
| 3 | 图表直接传达了 viewer_intent 的核心信息 |
| 2 | 需要用户主动分析才能获取核心信息 |
| 1 | 核心信息被次要信息淹没 |
| 0 | 图表完全无法传达 viewer_intent |

**技能 A 总分 = A1 + A2 + A3（满分 10）**

## 技能 B: 盲测解码 (Blind Decoder)

### 评分项

#### B1 — 5秒规则 (0–4分)

模拟"第一次看到此图，5秒内能否读出核心结论"：

| 得分 | 条件 |
|------|------|
| 4 | 核心结论立即可见，无需思考 |
| 3 | 5秒内能找到，需稍加引导 |
| 2 | 需要 10–20 秒仔细阅读 |
| 1 | 需要超过 30 秒分析 |
| 0 | 无法找到核心结论 |

#### B2 — 图表类型可辨度 (0–2分)

| 得分 | 条件 |
|------|------|
| 2 | 一眼识别图表类型，无歧义 |
| 1 | 需要看标题/标签才能确认类型 |
| 0 | 无法判断图表类型 |

#### B3 — 核心洞察可读性 (0–4分)

针对具体意图场景：

**预警场景**：
- 4分：预警数据点通过颜色/大小/位置明显区分
- 2分：预警只在数值上体现，视觉无强调
- 0分：无法识别哪些是预警数据

**对比场景**：
- 4分：最高/最低/差异值 清晰可见
- 2分：需要对比才能找到差异
- 0分：各项数据视觉权重相同，无法快速比较

**趋势场景**：
- 4分：趋势方向 + 转折点 一眼可见
- 2分：整体趋势可见，细节不清晰
- 0分：趋势被噪声淹没

**技能 B 总分 = B1 + B2 + B3（满分 10）**

## 技能 C: 认知摩擦力 (Friction Analyzer)

认知摩擦力越低越好。最终分数 = **10 - (摩擦力惩罚)**。

### 噪声源识别清单

#### 装饰性元素（每项 +0.05 friction）
- [ ] 3D 效果
- [ ] 渐变填充（无数据编码意义）
- [ ] 装饰性图标/背景图
- [ ] 过多网格线（超过 5 条）
- [ ] 不必要的阴影/发光效果

#### 色彩问题（每项 +0.1 friction）
- [ ] 色彩对比度低于 WCAG AA 标准（4.5:1）
- [ ] 使用颜色区分超过 7 个系列（色觉混淆风险）
- [ ] 红绿色盲不友好配色
- [ ] 色彩编码与数据含义不匹配（如冷色表示高温）

#### 信息过载（每项 +0.1 friction）
- [ ] 单图超过 5 条折线（Spaghetti Chart）
- [ ] 单图超过 12 个类别未分组
- [ ] 同屏超过 7 种不同图表类型
- [ ] 标注文字过多，遮挡数据

#### 布局混乱（每项 +0.08 friction）
- [ ] 图例位置不固定或难以关联
- [ ] 坐标轴未对齐
- [ ] 不一致的边距/间距
- [ ] 标题/副标题缺失或不清晰

### 摩擦力等级对应表

| Friction Score | 等级 | 含义 | 评测惩罚 |
|---------------|------|------|---------|
| 0.0 – 0.2 | ⭐ 极低 | 设计优秀，几乎无噪声 | 0 分 |
| 0.2 – 0.4 | 🟢 低 | 良好，轻微可改进 | -0.5 分 |
| 0.4 – 0.6 | 🟡 中 | 可接受，有明显改进空间 | -1.5 分 |
| 0.6 – 0.8 | 🟠 高 | 影响理解，需要改进 | -3.0 分 |
| 0.8 – 1.0 | 🔴 极高 | 严重干扰信息传达 | -5.0 分 |

## 综合评分计算公式

```
技能A分 = (A1×0.3 + A2×0.4 + A3×0.3) × 10
技能B分 = (B1×0.4 + B2×0.2 + B3×0.4) × 10
技能C分 = 10 - friction_penalty

综合分 = 技能A分 × 0.30
       + 技能B分 × 0.35
       + 技能C分 × 0.35
```

### 权重说明

- **技能B(35%)** 权重最高，因为视觉感知是可视化的核心目的
- **技能C(35%)** 同等重要，认知负担直接影响传播效果
- **技能A(30%)** 作为结构性基准，确保图表类型选择正确

## 边界情况处理

### 信息不足时
- 用户只提供图表截图：执行技能B和C，技能A基于视觉判断
- 用户只提供代码：执行技能A和C，技能B需描述性推理
- 用户只描述图表：仅执行技能A，给出建议但无法完整评分

### 特殊图表类型
- **自定义/实验性图表**：无预设期望拓扑，重点评技能B和C
- **艺术性信息图**：降低A维度权重，提高B维度权重
- **交互式图表**：额外评估交互设计的认知负担
